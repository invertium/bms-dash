Fixes the password prompt for protected packs never appearing: 1.2.0 only
asked for a password if it already had one saved, so a locked pack looked
like a dead device ("Waiting for battery data...", then disconnect). The
app now probes every pack on connect, exactly like the vendor app does,
and prompts when the pack says it is protected.

Unprotected packs are unaffected beyond a brief probe on connect.
